# Release Notes

## v0.1.0 (24 декабря 2025)

Первый релиз системы мониторинга атмосферных параметров для встраиваемых систем на базе ARM.

### Текущие возможности

**Аппаратная поддержка**:
- SCD41 (I2C 0x62): опрос CO₂ (400-5000 ppm), температуры (-10...+60°C), влажности (0-100% RH)
- BME280 (I2C 0x76/0x77): измерение атмосферного давления (300-1100 hPa), температуры, влажности
- SSD1306 (I2C 0x3C/0x3D): вывод на OLED-дисплей 128×64 с масштабированием шрифта (3×) и жирным начертанием
- DS3231 (I2C 0x68): опциональный RTC-модуль

**Функциональность**:
- Асинхронное чтение датчиков через `std::async` с параллельной обработкой данных
- Фоновое логирование в CSV через producer-consumer очередь (формат: timestamp, CO₂, T, P, RH)
- Интеллектуальное обновление дисплея (перерисовка только при изменении значений ≥1%)
- Self-test режим с JSON-выводом для автоматизированной проверки периферии
- Корректная обработка системных сигналов (SIGINT/SIGTERM) с graceful shutdown

**Разработка и сборка**:
- Кросс-платформенная сборка: host build (x86_64 с mock-датчиками) и target build (ARM с Buildroot SDK)
- Docker-образ для CI/CD с multi-stage сборкой
- CPack для генерации `.tar.gz` артефактов
- Модульная архитектура с разделением драйверов, соединений и бизнес-логики

# Release Notes

## v0.2.0 (27 декабря 2025)

### Улучшения

**Зависимости без Boost**:
- Реализованы встроенные парсеры JSON и опций командной строки без зависимости от Boost
- Если Boost не найден в SDK, приложение автоматически использует fallback реализацию
- Снижен порог совместимости: минимум C++20 (вместо C++23)
- Успешная сборка на ARM SDK с компилятором GCC 10.2.0

**Портативность**:
- CMake опция `USE_BOOST` для явного управления использованием Boost
- Добавлена поддержка pthread для cross-platform многопоточности
- JSON парсер поддерживает стандартный формат конфигов

### Технический стек

- C++20 (std::span, std::optional, встроенные парсеры)
- CMake ≥3.18
- Boost ≥1.70 (опциональная зависимость)
- I2C userspace driver через `/dev/i2c-*`

### Установка

```bash
tar -xzf atmolyt-host-0.1.0-Linux.tar.gz
cd atmolyt-host-0.1.0

# Развертывание как System V init сервис
sudo ./scripts/install.sh

# Альтернативно: ручной запуск
./scripts/start.sh
```
